5.2.2. 注入List集合类型的值

假设类中存在List集合类型的属性需要被注入值，例如：

x
// JavaSE, MySQL, Spring
private List<String> skills;

在配置时，应该在<property>的子级添加<list>节点，并在<list>的子级添加若干个<value>节点来配置值：

x
<property name="skills">
  <list>
    <value>JavaSE</value>
    <value>MySQL</value>
    <value>Spring</value>
  </list>
</property>

5.2.3. 注入Set集合类型的值

为Set集合注入值的操作方式与List集合的操作方式高度相似。

假设有Set集合的属性需要被注入值：

x
// Beijing, Shanghai, Guangzhou, Hangzhou, Shenzhen
private Set<String> cities;

则可以配置为：

x
<property name="cities">
  <set>
    <value>Beijing</value>
    <value>Shanghai</value>
    <value>Guangzhou</value>
    <value>Hangzhou</value>
    <value>Shenzhen</value>
  </set>
</property>

5.2.4. 注入Map集合类型的值

假设有Map集合的属性需要被注入值：

x
// username=Billy, password=1234, age=28
private Map<String, String> session;

需要配置为：

x
<property name="session">
  <map>
    <entry key="username" value="Billy"></entry>
    <entry key="password" value="1234"></entry>
    <entry key="age" value="28"></entry>
  </map>
</property>

5.2.5. 注入数组类型的值

假设有数组类型的属性需要被注入值：

xxxxxxxxxx
// 9, 5, 2, 7
private int[] numbers;

需要配置为：

x
<property name="numbers">
  <array>
    <value>9</value>
    <value>5</value>
    <value>2</value>
    <value>7</value>
  </array>
</property>

5.2.6. 注入Properties类型的值

通常，关于项目的配置，都会写在专门的.properties文件中，假设存在jdbc.properties文件，其配置信息是：

xxxxxxxxxx
url=jdbc:mysql://localhost:3306/db_name
driver=com.mysql.jdbc.Driver
username=root
password=1234

可以在项目的某个类中，声明Properties类型的属性，通过Spring框架读取jdbc.properties，并将读取到的配置信息注入到这个Properties类型的属性中！

则需要先在项目中添加Properties类型的属性：

x
// 值来自jdbc.properties文件
private Properties jdbc;

然后，在Spring的配置文件中，专门使用<util:properties>节点读取以上jdbc.properties，该节点就是专门用于读取.properties文件的，且该节点一旦被Spring读取，就会直接读取jdbc.properties文件，然后，Spring框架会把读到的信息封装为一个Properties类型的对象！

所以，在Spring的配置文件中添加：

x
<!-- util:properties节点：专门用于读取.properties配置文件的，其本身相当于一个bean节点，被加载后，就是一个Properties对象 -->
<!-- location属性：需要读取的配置文件的位置 -->
<util:properties id="jdbc2" location="classpath:jdbc.properties"></util:properties>

需要注意：<util:properties>节点与以前使用的<bean>节点都是直接归属于根级的<beans>节点的，千万不要把<util:properties>节点写到<bean>的子级了！

最后，当需要注入属性的值时，使用<property>节点的ref属性引用到以上<util:properties>节点：

x
<!-- ref属性：当属性的值是另一个Bean时，使用ref属性来配置，取值就是那个Bean的id -->
<property name="jdbc" ref="jdbc2"></property>

所以，在注入属性值时，如果属性的值是基本值（字符串、数值、布尔值），是可以直接书写出来的，就使用value属性来配置，如果属性的值是另一个Bean，则使用ref属性来配置，且ref属性的值是另一个Bean的id。

另外，其实也可以在Spring的配置文件中，使用类型注入List集合的方式直接将配置信息写在XML中进行注入：

x
<property name="jdbc">
  <props>
    <prop key="initialSize">10</prop>
    <prop key="maxActive">50</prop>
  </props>
</property>

因为一般配置信息都推荐写在.properties文件中，所以，重点还是掌握<util:properites>的使用方式。

5.2.7. 小结

当需要通过Spring框架为属性注入值时：

  • 属性必须有规范名称的SET方法；
  • 在Spring的配置文件中，使用<property>节点注入属性的值，该节点的name属性就是需要注入值的属性名；
  • 在<property>节点中，如果属性值是基本值（字符串、数值、布尔值），使用value属性配置其值，如果属性值是另一个Bean，则使用ref属性引用另一个Bean的id；
  • 当需要注入的值是各种集合类型（List / Set / Map / 数组 / Properties）时，都可以添加对应的子级节点（<list> / <set> / <map> / <array> / <props>）节点进行配置，也可以使用<util:list>等节点配置好各集合对象，然后再通过ref属性进行引用；
  • 需要掌握使用<util:properties>节点读取.properties文件的做法；

5.3. 通过构造方法注入属性的值

假设存在User类，其中的name属性，需要通过构造方法为该属性注入值，则需要添加带参数的构造方法，并且，可以通过构造方法的参数为属性赋值：

xxxxxxxxxx
public class User {
  
  // Kate
  private String name;
​
  public User(String name) {
    super();
    this.name = name;
  }
​
  @Override
  public String toString() {
    return "User [name=" + name + "]";
  }
​
}

在配置时，需要在<bean>节点下通过<constructor-arg>节点配置构造方法的参数：

x
  <!-- 通过构造方法注入属性值 -->
  <bean id="user" class="cn.tedu.spring.User">
    <!-- constructor-arg节点：配置构造方法中的参数 -->
    <!-- index属性：当前配置的是构造方法中的第几个参数 -->
    <constructor-arg index="0" value="Kate"></constructor-arg>
  </bean>

如果类中有多个属性，并且多个属性都需要通过构造方法注入值：

xxxxxxxxxx
public class User {
​
  // Kate
  private String name;
  // Shanghai
  private String from;
  // 26
  private int age;
​
  public User(String name, String from, int age) {
    super();
    this.name = name;
    this.from = from;
    this.age = age;
  }
​
  @Override
  public String toString() {
    return "User [name=" + name + ", from=" + from + ", age=" + age + "]";
  }
​
}

在配置时，在同一个<bean>节点添加多个<constructor-arg>节点来配置多个值即可：

xxxxxxxxxx
  <!-- 通过构造方法注入属性值 -->
  <bean id="user" class="cn.tedu.spring.User">
    <!-- constructor-arg节点：配置构造方法中的参数 -->
    <!-- index属性：当前配置的是构造方法中的第几个参数 -->
    <constructor-arg index="0" value="Kate"></constructor-arg>
    <constructor-arg index="1" value="Shanghai"></constructor-arg>
    <constructor-arg index="2" value="26"></constructor-arg>
  </bean>

6. Spring表达式

在Spring的配置文件中，当需要注入值时，可以通过Spring表达式获取另一个Bean中的某个属性的值！例如在Spring容器中，有一个叫teacher的对象，其中有个属性class的值为jsd1911，需要配置另一个Student类，在Student类中也有名为class的属性，如果希望这个属性值就是teacher的class值，也就是jsd1911这个值，就可以通过Spring表达式来进行配置。

Spring表达式的基本语法格式是：

x
#{bean的id.属性名}

例如：

x
  <bean id="valueBean" class="cn.tedu.spring.ValueBean">
    <property name="name" value="#{sampleBean.name}"></property>
  </bean>

另外，还可以通过Spring表达式读取List集合类型的属性值中的某1个值，其语法格式是：

x
#{bean的id.List集合的属性名[索引位置]}

通过这种方式，也可以获取另一个Bean中的数组类型的对象中的某个值，甚至，还可以获取Set集合中的某个元素的值！

如果需要读取的是Map或Properties类型的集合中的某1个值，其语法格式是：

x
#{bean的id.Map集合的属性名.key}

或者，也可以使用以下语法：

xxxxxxxxxx
#{bean的id.Map集合的属性名['key']}

7. 自动装配

自动装配是Spring框架中实现性极高的一个机制！当某个属性需要值时，Spring框架会在整个Spring容器中找到符合的值，并自动为这个属性赋值，在整个过程中，开发人员不需要显式的使用SET方式或构造方法为属性注入值，全程由Spring框架自动完成即可！

在<bean>节点中添加autowire属性就可以实现自动装配，该属性的常用取值有byName和byType。

当取值为byName时，表示按照名称实现自动装配，也就是说：Spring框架会从Spring容器中查询Bean Id与属性名（其实是SET方法名）一致的对象，如果找到了，就自动的为属性赋值！

当取值为byType时，表示按照类型实现自动装配，也就是说：Spring框架会从Spring容器中查询类型匹配的对象，如果找到了，就自动的为属性赋值！如果匹配类型的对象超过1个，就会装配失败，则不应该使用这种模式来实现自动装配。

使用这种做法实现自动装配存在很大的问题，当一个类中有多个属性，且这些属性都有SET方法时，也许，其中的某些属性并不希望通过自动装配来赋值，当使用byName或byType装配时，它们都会根据装配规则尝试装配所有属性，最终，多个属性中，哪些被成功的装配了值，而哪些并没有被装配，或者，希望被装配的是否都有值，而不希望装配的是不是都没有值，这一点，不便于查看的！所以，会导致代码非常不直观！所以，在实际应用中，并不会通过这种方式来实现自动装配，而是通过注解来实现。

所以，目前主要理解byName和byType的装配模式即可。

附：关于List与Set集合

List与Set都是Collection的子级接口。

List集合是序列的，并且集合中的元素是可以重复的，即：如果向集合中反复添加相同的元素，最终集合中就会有多个相同的元素；

Set集合是散列的，并且集合中的元素是不可以重复的，即：如果向集合中反复添加相同的元素，所调用的add()方法会返回false，最终集合中并不会出现相同的元素。

在集合中，判断2个对象是否相同，取决于它们的hashCode()值是否相同，并且，equals()对比的结果为true。

其实，每一个Set都是一个不关心Value的Map集合。

注意：不要说Set是无序的，这种说法并不严谨，通常，说Set无序，是因为遍历Set时，看到的各元素的顺序是没有规律的，其实，这取决于Set 的实现类，如果使用的是HashSet，则遍历结果就是无序的，如果使用的是TreeSet，则遍历结果是自然排序（根据元素的类型实现Comparable接口来决定）的，如果使用的是LinkedHashSet，则遍历结果是根据添加时的顺序来排列的！
